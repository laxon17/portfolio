Clear code

For me, clear written code is the most important thing in an app. It shows your meticulousness, that you think about your colleagues and that they don't need to search code lines for centuries.
As every web developer, I use Git and GitHub too. It helps me to store and track code changes in my repo. With Git you can learn new things or fulfill your previous knowledge in coding and developing, because more experienced developers can watch and give you advices...

Keep in mind

  • The code should be easy to be read,
  • Use identations and comments for each element,
  • Follow a specific method for commenting on the work,
  • Name the functions according to what they perform,
  • The language functions that are complex or the structure that is difficult to be comprehended should be avoided
  • Use Git and GitHub.
